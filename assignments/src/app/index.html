Rendering concepts


Server-Side Rendering (SSR)

Server-Side Rendering (SSR) is a technique used in web development where the server generates the HTML content of a page before sending it to the client's browser. This differs from Client-Side Rendering (CSR) , where the browser downloads a minimal HTML shell and populates the content dynamically using JavaScript. In SSR, the server compiles and renders the webpage on request and sends the fully rendered HTML to the client.

Benefits of SSR

  • Improved SEO: Since search engines can directly crawl fully rendered HTML, SSR improves the visibility of the content in search engine results.
  • Faster initial load: The initial load time is faster, as users don't need to wait for JavaScript to fetch and render the content.
  • Better performance on low-powered devices: Since rendering happens on the server, it can be more efficient for devices with lower processing power.

Use Cases of SSR

  • When SEO is critical (e.g., for e-commerce sites or blogs).
  • When the app needs to render complex data quickly on the first page load.

Client-Side Rendering (CSR)

Client-Side Rendering (CSR) is a web development technique where the content of a webpage is generated on the clientâ€™s side (the browser) rather than the server. Here's how it works:

How CSR Works

  • Initial Request: When a user accesses a website, the server sends a basic HTML document and JavaScript files to the browser.
  • Rendering in the Browser: The JavaScript executes in the browser, fetching additional content (often from an API) and dynamically updating the DOM to display the full page.
  • Updates and Interactions: Subsequent updates to the page (like clicking a link or submitting a form) often don't require a full page reload. JavaScript can handle this within the browser using frameworks like React, Angular, or Vue.js.

Advantages of CSR

  • Fast User Interactions: After the initial page load, interactions are generally faster since only specific parts of the page are updated.
  • Rich User Interfaces: CSR allows for more dynamic, responsive interfaces by leveraging JavaScript directly in the browser.

Disadvantages of CSR

  • SEO Challenges: Since the content is loaded dynamically, search engine crawlers may not index the page content as easily (though this can be mitigated with server-side rendering or static generation).
  • Longer Initial Load Time: The initial page load can be slower, especially if the JavaScript bundle is large.

CSR is commonly used in single-page applications (SPAs) where a fluid, app-like experience is needed.

SSG (Static Site Generation)

In SSG, HTML files are generated at build time. This means that when you build your website, all the pages are pre-generated as static HTML files. When a user visits a page, they see the pre-built static HTML, resulting in faster load times.

Key Features:

  • SEO-friendly because pages are pre-rendered for search engines.
  • Better performance as there is no need to fetch dynamic data on each request.
  • Data is fetched and rendered during the build process, not at runtime.


ISR (Incremental Static Regeneration)

ISR allows you to dynamically update static pages by regenerating them in the background after a certain interval or trigger. It provides the performance benefits of static pages while keeping content up to date.

Key Features:

  • Static pages are served, but content can still be updated periodically.
  • No need to rebuild the entire website, only individual pages are revalidated.
  • You can control how frequently a page is revalidated (e.g., every 10 seconds).
